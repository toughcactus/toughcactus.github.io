虚幻引擎 > 进阶课程 > 虚幻编辑器 > 1.10.1实现转向动画

  • 当前实现了3种动作，走路，跑步和跳跃

  • 走路这块，从静止到跑起来支持加速，使用BS_Locomotion

    • 本小节会重新实现混合空间，根据速度方向，加载转向和后退动画
  • 之后还有装弹(Reloading)，死亡(Death)，瞄准(Aiming)动画

  • Actor有ForwardVector和RightVector，这个是TurnAround回调函数中会修改的

    • 当我们水平移动鼠标，ForwardVector和RightVector随之发生改变
    • 在视觉上，ForwardVector一直向前，RightVector一直向右，二者垂直

  • Actor的速度是一个矢量，其表现是单位时间内的位移方向

    • 当速度不为0时，VelocityVector和ForwardVector的夹角与此时Actor应该展现的动画息息相关

  • 点乘

    • 其中一个向量为0向量时，点乘结果为0，而反余弦计算得到其夹角为PI/2

  • 叉乘

    • c.z(向量的z坐标)的符号说明向量c的方向

    • theta为0，PI和-PI时，叉乘结果为0 > 无法通过向量确定平面，也就无法得到平面法向量

  • 夹角的计算通过反余弦结果和sign(c.z)的乘积实现，分析特殊情形

    • 坐标 ( Velocity, Angle )

    • 速度为0，计算夹角得到PI/2，由于此时Velocity为0，( 0, 任意 ) 对应Idle

      • ( 600, PI/2 ) 对应满速向右

    • 速度不为0，夹角为0或PI，sign(c.z)为0

      • 此时乘积为0，只能使用RunFwd

      • 夹角为0使用Fwd正确，夹角为PI应该使用Bwd

        • 夹角为PI时，不使用乘积，使用反余弦结果
      夹角 反余弦结果 理论上的sign(c.z) 目的动画 使用乘积效果
      0 0 0 Fwd o
      ( 0, PI/2 ) ( 0, PI/2 ) 1 Fwd > Right o
      PI/2 PI/2 1 Right o
      ( PI/2, PI ) ( PI/2, PI ) 1 Right > Bwd o
      PI PI 0 Bwd x
      -PI PI 0 Bwd x
      ( -PI, -PI/2 ) ( PI/2, PI ) -1 Left > Bwd o
      -PI/2 PI/2 -1 Left o
      ( -PI/2, 0 ) ( 0, PI/2 ) -1 Fwd > Left o
  • 可以在蓝图中实现夹角的计算，也可以在C++中实现

  • Blend Space

    • 2D混合空间
    • 两个轴，vertial axis和horizontal axis

一、绘制ForwardVector，RightVector和VelocityVector

  • 使用DrawDebugArrow

    • Duration为0，每帧都进行绘画

绘制ForwardVector

  • 起点 Actor当前位置
  • 偏移 放大的前进方向向量
  • 终点 起点 + 偏移

绘制RightVector

  • 起点 Actor当前位置
  • 偏移 放大的向右方向向量

绘制VelocityVector

  • 起点 Actor当前位置
  • 偏移 放大的速度方向单位向量

效果图

  • 一直显示ForwardVector和RightVector

  • 有速度才会显示VelocityVector

二、优化显示

  • 项目设置 > Engine > Rendering > Default Settings > Auto Exposure
  • 旨在模拟人眼适应不同亮度

  • 取消勾选

    • 看不出区别

三、重新实现Walk动画

  • 使用Blend Space，命名为BS_Locomotion_Walk

  • 水平轴(horizontal axis)和1D一样，表示速度

    • 命名为Velocity
    • [ 0, 600 ]

  • 垂直轴(vertical axis)，表示ForwardVector和VelocityVector的夹角

    • 命名为Direction
    • [ -180, 180 ]

  • 坐标 ( Velocity, Direction )

1. 动画资产

  • Idle
  • Run_Fwd
  • Run_Bwd
  • Run_Rt
  • Run_Lt

2. 初始状态 ( 0, 0 )

  • Idle

3. 满速前进 ( 600, 0 )

  • Run_Fwd

4. 满速后退 ( 600, -180 ) 和 ( 600, 180 )

  • Run_Bwd

5. 满速向右 ( 600, 90 )

  • Run_Rt

6. 满速向左 ( 600, -90 )

  • Run_Lt

四、将BS_Locomotion_Walk作为ABP_BaseCharacter > AnimGraph > 状态机 > Walk状态的输入

  • 添加变量 AngleBetweenForwardAndVelocity

    • float类型
    • 作为Direction的输入

    • 默认值为0

五、在蓝图中计算夹角并设置变量

  • 计算点乘时，ForwardVector和VelocityVector都是单位向量
  • 当前并未在夹角为PI时做处理，后退动画偶有闪动但看似正常

1. 输出DotProduct结果，静止时输出90

2. 输出CrossProduct结果，静止、前进或后退时，坐标为0但是有正负

3. 输出Sign结果，静止时为0，前进或后退时，其结果或为1或为-1，飘忽不定，但不为0，所以后退动画看似正常

  • 可以做出如下推测 > 显示为0但不意味着为0，数值过于小了但是有正负
  • 叉乘时传入未单位化的Velocity无改善

4. 同时按下后退和向左，或者后退和向右，正负3PI/4的动画较为刻意

  • ( PI/2, PI ) 和 ( -PI, -PI/2 ) 时可以只显示Run_Bwd

六、绘制ForwardVectro和VelocityVector的叉乘结果

  • 起点 Actor当前位置
  • 偏移 放大的叉乘结果
  • 运动时才有向量显示
  • 向前和向后运动时，偶有不明显的显示
