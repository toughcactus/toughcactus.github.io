一、代码规范

  • 按个人习惯来，非建议
  • 参考

  1. 大括号

    • 放在新行

  2. 缩进

    • 虚幻引擎建议使用制表符，也可以使用空格

二、.clang-format文件

  • 很多项目都包含该文件，描述了管理代码文件的统一格式
  • 我们可以通过该文件设置格式化代码的规则
  • YAML标记文件
  • 在ShootTraining中添加.clang-format文件
  • 每行遵循格式 key: value
  • 参考

1. 设置应用该格式的语言

  • 如果项目中只一种开发语言，或者所有开发语言使用相同规范，可忽略该配置
  Language: Cpp

2. Style

  • Google

    • 一个制表符对应2个空格，大括号不在新行

        BasedOnStyle: Google

  • Microsoft

    • 一个制表符对应4个空格，大括号在新行

        BasedOnStyle: Microsoft

3. 缩进宽度 

  IndentWidth: '4'

4. 缩进使用制表符或空格

  • 缩进使用空格

      UseTab: Never

  • 缩进使用制表符

    • 源文件中既有空格又有制表符
      UseTab: Always

5. 制表符宽度

  • 每6个空格替换一个制表符，当缩进使用制表符时，效果很糟糕

      TabWidth: '6'

6. 设置大括号位置

  • 不在新行

      BreakBeforeBraces: Attach

  • 在新行

      BreakBeforeBraces: Allman

7. 其他设置 

  ColumnLimit: '140'
  AccessModifierOffset: '-4'
  SortIncludes: false
  AllowShortBlocksOnASingleLine: false
  AlignAfterOpenBracket: DontAlign
  AllowShortFunctionsOnASingleLine: Inline
  PointerAlignment: Left
