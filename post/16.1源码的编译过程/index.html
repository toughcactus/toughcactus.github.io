一、引擎代码模块化

  • EpicGames/UE_5.1/Source/
  • 引擎代码由若干个模块组成，模块对指定功能进行了封装，如AI、特效和用户界面等
  • 每个模块拥有一个xx.Build.cs文件，包含其编译说明
  • 模块类的编译规则拥有基类 ModuleRules，我们可以对编译规则进行设置
  • 不同平台上，模块的编译规则也不同

二、TheBasics.Build.cs文件中的编译规则

  1. 依赖模块列表 PublicDependencyModuleNames

    • 4个默认依赖

      • Core
      • CoreUObject
      • Engine
      • InputCore
  2. 可以添加要使用的第三方库，如机器学习库和计算机视觉库

  3. 可用构建选项

    • 参考

三、TheBasics模块

  • Souce/TheBasics/TheBasics.cpp中，使用宏定义了游戏模块
  • 游戏也可以由模块组成

四、项目描述 TheBasics.uproject

  • json格式
  • Mudules字段描述游戏模块TheBasics
  • EngineAssociation字段描述引擎版本
  • 虚幻引擎根据uproject生成项目所需文件，并根据虚幻编辑器的设置来运行项目

五、预编译头文件(PCH, Precompiled Headers)机制

  1. 预编译头文件机制提前处理模块内文件，并替换相应文本，可以减少编译项目时预处理器的工作量

  2. 模块预设共享头文件

    • 以Engine/Source/Runtime/Engine/为例

    • Engine.Build.cs文件中

      • 包含模块相应头文件

  3. TheBasics模块使用预编译头文件

    • Source/TheBasics.Build.cs文件

    • PCHUsage项默认设置

      • 编译时，会包含依赖模块提供的SharedPCHHeaderFile

  4. 不使用预编译头文件机制

    • 设置PCHUsage为NoPCHs

        // PCHUsage = PCHUsageMode.UseExplicitOrSharedPCHs;
  PCHUsage = PCHUsageMode.NoPCHs;
    • 按照之前添加的头文件，编译没有问题
    • 换言之，使用默认PCHUsage设置，可以不用添加那些头文件
    • 注意，未包含USceneComponent的头文件，AActor中有包含
