C++ > 动态内存 > allocator

  • allocator对象的4个操作

    • 申请未构造的动态内存 ｜ allocate
    • 有序构造动态对象 ｜ construct
    • 逆序析构动态对象 ｜ destroy
    • 释放动态内存 ｜ deallocate

一、默认初始化

  • 创建一个allocator对象
  allocator<T> a;

二、allocate | 申请动态对象个数为n的动态内存

  • 动态内存是原始的、未构造的
  • 返回值为指向这片动态空间的内置指针
  • 保存指针和对象个数
  T *p = a.allocate(n);

三、deallocate | 释放allocate申请的动态内存

  • p为allocate的返回值
  • n为allocate给定的动态对象个数
  • p、n不具有灵活性
  • 在调用deallocate之前，保证对象已被析构
  a.deallocate(p, n);

四、construct | 使用参数匹配构造函数，在p指向的位置构造动态对象 

  a.construct(p, args);

五、destroy | 对p所指向的对象调用析构函数 

  a.destroy(p);

六、使用allocator 

  allocator<string> alloc;
  auto n = 10;
  auto start = alloc.allocate(n), cur = start; // 保存start，调用deallocate时使用；cur指向未构造对象

  alloc.construct(cur++); // 空串
  alloc.construct(cur++, 10, 'c'); // 10个'c'
  alloc.construct(cur++, "hi"); // "hi"

  while (cur != start)
      alloc.destroy(--cur);// 挨个构造，挨个析构

  alloc.deallocate(start, n);

七、初始化allocator管理的动态内存或为其赋值

1. uninitialized_copy | 使用迭代器范围[b, e)初始化allocator管理的动态内存

  • cur指向待构造对象
  • 默认动态空间能容纳
  • 返回指针指向待构造对象
  uninitialized_copy(b, e, cur);

2. uninitialized_copy_n | 使用迭代器范围[b, b + n)初始化allocator管理的动态内存

  • b为迭代器起点
  • n为个数
  • cur为待构造对象
  • 返回指针指向待构造对象
  uninitialized_copy_n(b, n, cur);

3. uninitialized_fill | 为迭代器范围[b, e)中的动态对象赋值

  • 返回类型为void
  uninitialized_fill(b, e, t);

4. uninitialized_fill_n | 为迭代器范围[b, b + n)中的动态对象赋值

  • 返回指针指向第(b+n)个对象
  uninitialized_fill_n(b, n, t);
