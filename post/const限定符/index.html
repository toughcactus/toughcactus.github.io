C++ > 类型 > const限定符

也是类型修饰符。

采用《C++ Primer》中的概念。

和常量指针，指针常量，常量引用这些概念相冲突。

  • 常量指针(pointer to const) ｜ 指向常量的指针
  • 指针常量(const pointer) ｜ 指针是常量
  • 常量引用(const reference) ｜ 引用常量

一、顶层const

  • 只有对象能拥有顶层const属性

    • 左值引用不是对象，不能拥有顶层const
  • 拥有顶层const的对象只读，必须显式初始化
  • 描述对象自身的读写权限
  const T var = val;

二、const指针

  • 具有顶层const属性的指针
  • 不能对const指针进行写操作，即不能使其指向其他对象
  • 关联对象的读写权限与此无关
  T var, *const cp = &var;

三、底层const

  • 基于间接访问，只有左值引用和指针可以拥有底层const属性

  • 如果指针或引用拥有底层const，对于二者而言，关联对象只读

    • 关联对象不一定具有顶层const，但对引用或指针锁定写权限
  • 描述关联对象读写权限的开放情况
  • 指针的关联关系与此无关

1. 使用指针对指针进行赋值时，底层const属性被传递，要求左右两侧运算对象的底层const一致

2. 初始化引用和指针时，如果关联对象有顶层const，要求引用和指针有底层const 

  int a = 5, *p = &a;
  const int *const cpc = p; // 顶层const + 底层const
  const int b = 6;
  const int *pc = &b; // 底层const

  int *p2 = pc; // 错误：p2无底层const，而b不允许被修改
  int &ri = b; // 错误：ri无底层const，而b不允许被修改

四、具有底层const的指针 

  const T *pc;

五、const引用

  • 具有底层const的左值引用
  int i = 5;
  const int &rci = i; // 关联对象锁定写权限
  i = 6;
  cout << rci<< endl; // 6
  cout << &rci<< endl << &i << endl; // 二者地址相同

可以绑定右值

  • 初始值为表达式，或存在隐式类型转换，是纯右值表达式
  • 纯右值表达式物质化得到速亡值表达式，对应一个明确对象，将引用绑定到该对象
  • 该对象对const引用锁定写权限
  double dval = 3.14;
  const int &rci = dval;
  cout << rci << endl; // 3

  /*   <=>
  const int temp = dval;
  const int &rci = temp;
  */

  cout << &dval << endl << &rci << endl; // 二者地址不同

六、具有底层const的右值引用

  • 该对象对底层const右值引用锁定写权限
  #include <iostream>
  int fcn(int a) { return a + 1; }
  int main()
  {
      int a = 5;
      const int &&r = fcn(a);
      //  r = 20; // 错误
      return 0;
  }

七、总结

1. 指针可以拥有顶层const和底层const；引用只能拥有底层const

2. 所有类型对象都可以拥有顶层const，但只有指针和引用可以拥有底层const

3. 定义具有底层const的const指针 

  const int *p; // 具有底层const的指针
  int *const cp = p; // const指针，具有顶层const
  const int *const cpc = p; // const指针，亦具有底层const

八、const属性和作用域

  • 在函数外声明的顶层const对象，具有文件作用域
  • 在函数外声明的底层const对象，具有全局作用域

1. 创建具有全局作用域的顶层const对象

  • 声明或定义顶层const对象时，使用extern关键字，使之具有全局作用域
  • 包含显式初始化的声明即定义，只能在一处定义

2. 具有文件作用域的顶层const对象和具有全局作用域的顶层const对象，因为作用域不同，可以出现在同一个文件中
