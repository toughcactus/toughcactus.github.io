C++ > 类型 > 指针和引用

一、绑定

  • 令某个名字与给定的实体（对象）关联在一起，使用该名字即使用绑定的实体（对象）
  int i = 5, &ri = i;
  cout << &i << endl << &ri << endl;  // 二者地址相同

二、指针和引用

  • 可以通过指针和引用间接访问对象

1. 是复合类型，使用类型修饰符声明引用或指针，使用基本类型描述关联对象的类型

* 指针
& 左值引用
&& 右值引用

2. 描述和对象的关联关系

  • 指针对象保存关联对象的地址，并指示关联对象的类型，形容为指向该对象
  • 引用绑定对象，是对象的别名

3. 指针是具体类型，引用不是

三、指针 

  T var, *p = &var; // 类型修饰符*说明p是一个指针，T为关联对象类型
  *p; // 返回关联对象的引用
  • 是对象
  • 类型修饰符*的前半部分描述关联对象，后半部分描述指针
  • 固定大小，和机器字长一致

1. 定义空指针

  • 不指向任何对象
  • 不允许对空指针解引用或访问成员 | 编译器报错
  int *p3 = nullptr; // 推荐使用
  int *p4 = 0; // 定义为空指针

  // #include <cstdlib> // NULL是预处理变量，不属于命名空间std；避免使用
  int *p5 = NULL;

2. 定义多个指针

  • 要求基本类型一致，每个指针都要使用类型修饰符*

      int *ip1, *ip2; // *ip1构成一个声明符，*ip2构成另一个声明符，int是基本类型

3. 取地址运算符&（单目） | 获取某个对象的地址

4. 解引用运算符*（单目） | 获取关联对象的引用

5. 成员访问运算符->（双目）| 获取关联对象数据成员的引用

6. void *指针

  • 可以存放任意类型对象的地址
  • 不能直接操作void *指针关联的对象，对象类型未知
  void *p;

四、引用

  • 对象的别名
  • 有两种，左值引用和右值引用
  • 是变量，不是对象；不拥有内存空间，关联到绑定对象的内存空间
  • 引用对应一个明确对象
  • 声明时必须显式初始化，将标识符与对象进行绑定；绑定关系不可更改
  • 引用的基本类型和关联对象类型必须严格匹配
  • 对引用读写，实则读写其绑定的对象

五、左值引用

  • 绑定一个左值表达式

  • 使用左值引用可以避免拷贝带来的开销

    • 调用函数时，如果Pass-by-value，会调用拷贝构造函数创建实参的拷贝，而Pass-by-reference-to-const可以避免拷贝和修改关联对象
  T var, &refVar = var;
  #include <iostream>
  using std::cout;
  using std::endl;
  int &fcn(int &a)
  {
      return ++a;
  }
  int main() {
      int a = 5;
      int &r = ++fcn(a);
      return 0;  
  }

六、右值引用

  • 绑定右值表达式
  • 接管了引用对象的资源，之后可通过引用对该对象进行读写
  #include <iostream>
  int fcn(int a) { return a + 1; }
  int main()
  {
      int a = 5;
      int &&r = fcn(a);
      const int &cr = r;
      r = 20;
      std::cout << cr << std::endl;
      return 0;
  }
